# GENERATE.md — agent-rules 生成規格

這份文件是 **agent-rules 系統的唯一權威**。任何 agent（Claude、Codex、或其他）被要求「生成設定檔」時，**完整照這份做**。

---

## 0. 心智模型

- **source 是唯一真相，你（agent）是 compiler。**
- 你讀某個 scope 的 `agent-rules/` source，輸出符合**你自己平台慣例**的設定檔。
- 你絕不擅自加入 source 沒有的規則；你只做「重排版、依平台慣例改寫、拆檔、建索引」。

---

## 1. 兩個 scope

| scope | source | 你要產出的東西 |
|---|---|---|
| 全域 | `~/agent-rules/` | Claude → `~/.claude/CLAUDE.md`；Codex → `~/.codex/AGENTS.md` |
| 專案 | `<專案>/agent-rules/` | Claude → `<專案>/CLAUDE.md`；Codex → `<專案>/AGENTS.md`；以及 `<專案>/wiki/`、`<專案>/reference/` |

**不要在專案輸出裡重複全域內容。** Claude 與 Codex 原生會自動合併「全域設定檔 + 專案設定檔」，重複只會浪費 context 並造成不同步。

---

## 2. source 佈局規則（資料夾即分類）

讀 `agent-rules/` 時，依**位置**決定每個檔的命運。只有兩條通則：

1. **頂層 `.md` 檔** = **core** → 內容**內嵌**進根設定檔（`CLAUDE.md` / `AGENTS.md`）。
   - **保留檔名例外（不視為 core、不內嵌）**：`GENERATE.md`、`generate.md`、`README.md`、`CHANGELOG.md`。
     這些是給人看的說明或工具用檔（生成指令、專案說明、變更紀錄），不是給 agent 的規則。生成時一律略過，**不要**內嵌進輸出。
2. **任何子資料夾** = 按需材料（資料夾名稱不限——`wiki/`、`reference/`、`playbooks/`、`design-tokens/`… 都行）
   → 把該資料夾**原樣**拆成輸出端的**同名資料夾**（含巢狀結構），其中每個檔對應輸出一份；根檔針對每個資料夾放一段索引連結，並說明「何時該讀」。
   - `wiki/`、`reference/` 只是常見例子，**不是固定清單**；不要因為資料夾不叫這兩個名字就特殊處理。

> 為什麼分？根設定檔每次對話都被整個讀進 context，要**精簡**。常駐規則內嵌；大份材料拆出去、按需載入（progressive disclosure）。

---

## 3. 生成程序（逐步）

1. **確認 scope**：判斷在做全域還是某專案，據此決定 source 與輸出路徑（見 §1）。
2. **讀全部 source**：讀該 scope `agent-rules/` 底下所有 `.md`（含 `wiki/`、`reference/` 等子資料夾）。
3. **讀生成指令**：
   - 全域：本檔（`~/agent-rules/GENERATE.md`）即指令。
   - 專案：先讀 `<專案>/agent-rules/generate.md`；它會引用本檔為 base，再補/覆寫專案特有規則。**專案 generate.md 與本檔衝突時，以專案 generate.md 為準。**
4. **套用你的 agent profile**（見 §4），決定輸出檔名、位置、平台特有段落、語氣。
5. **組裝根設定檔**：
   - 把所有 core（頂層檔）內容，依**檔名排序**，依你平台慣例改寫後內嵌。
   - 末尾加一個「索引 / 延伸資料」段落，用**相對路徑連結**指向**每個子資料夾**裡的每個檔，並各寫一句「何時該讀它」。
6. **拆出細節檔**：把**所有子資料夾**原樣寫到輸出端對應的同名位置（含巢狀結構）。內容可依平台慣例微調，但不增刪規則。
7. **套用擁有區機制**（見 §5）：保留既有輸出檔裡的「手動區」。
8. **加 banner**（見 §5）到每個你產生的檔頂部。
9. **回報**：簡述你產生/更新了哪些檔、保留了哪些手動區、有沒有偵測到需反吐回 source 的手改（見 §6）。

## 4. Agent profiles

只記錄真正的差異。共通原則：根檔精簡、core 內嵌、細節拆檔加索引、不增刪 source 沒有的規則。

### Claude profile

- **輸出檔**：專案 `CLAUDE.md`（專案根）／全域 `~/.claude/CLAUDE.md`。
- **原生合併**：Claude Code 會自動載入 `~/.claude/CLAUDE.md`（user）+ 專案 `CLAUDE.md` + 子目錄的巢狀 `CLAUDE.md`。所以**專案檔不要重抄全域**。
- **連結方式**：用**一般 markdown 相對連結**指向 `wiki/`、`reference/`（要的是「按需才讀」）。**不要**用 `@path` import 語法指它們——`@import` 會強制把檔拉進 context，破壞 progressive disclosure。`@import` 只在你「真的想要某段每次都載入」時才用。
- **平台特有段落**：可提及 skills、hooks、slash commands、MCP（若 source 有相關規則）。沒有就不要無中生有。
- **語氣**：祈使句、精簡、條列。

### Codex profile

- **輸出檔**：專案 `AGENTS.md`（專案根）／全域 `~/.codex/AGENTS.md`。
- **原生合併**：Codex 會由內而外合併目錄樹上的 `AGENTS.md` 與全域 `~/.codex/AGENTS.md`。所以**專案檔不要重抄全域**。
- **連結方式**：用一般 markdown 相對連結指向 `wiki/`、`reference/`。
- **平台特有段落**：`AGENTS.md` 是純說明文件，沒有 Claude 的 skills/hooks 概念——對應內容請改寫成一般指示，或省略。
- **語氣**：直白、直接、條列。

> 未來新增 agent（Cursor / Windsurf 等）：在此新增一段 profile 即可，其餘機制不變。

## 5. 擁有區 vs 手動區（防 drift）

每個生成檔都區分兩種區域，**重生時行為不同**：

- **generator 擁有區**：由 source 產生，**重生會整段覆蓋**。預設整份檔都是擁有區。
- **手動區**：使用者直接在輸出檔寫的內容，**重生必須原樣保留、絕不更動**。

用 HTML 註解標記（在 markdown 不顯示、可被解析）：

```markdown
<!-- agent-rules:manual:start -->
這裡是使用者手寫、重生要保留的內容。
<!-- agent-rules:manual:end -->
```

未被 `manual` 包住的內容，一律視為 generator 擁有區。

**重生時的鐵則**：
1. 先掃描既有輸出檔，**完整抄下所有 `manual` 區塊**（含標記）。
2. 重新生成擁有區。
3. 把保留的 `manual` 區塊放回**原本相對位置**（盡量貼近原本前後文）。

### Banner

每個生成檔**頂部**放這段（依平台註解習慣，markdown 用 HTML 註解）：

```markdown
<!--
GENERATED by agent-rules — source: <該 scope 的 agent-rules/ 路徑>
- 優先改 source 再重生；不要直接改 generator 擁有區。
- agent-rules:manual 區塊內的內容，重生會保留。
- 若你已直接改了擁有區，重生前請先把改動反吐回 source（見 GENERATE.md §6）。
-->
```

---

## 6. 迭代回路（手改 → 反吐回 source）

允許使用者直接編輯生成檔。但**擁有區**的手改在下次重生會被覆蓋，所以：

- **重生前**，比對既有輸出檔的擁有區 vs 由 source 重新生成的結果。
- 若偵測到「擁有區被手改、但 source 沒有對應內容」：
  1. **不要默默覆蓋。** 先把這段改動**反吐回對應的 source 檔**（core 改動回頂層檔；細節改動回它原本所屬的子資料夾）。
  2. 在 source 該處或 `agent-rules/CHANGELOG.md` 記**一行理由**（為什麼這樣改、何時、來自哪個生成檔）。
  3. 再進行重生。
- 若使用者只是想加「不進 source 的一次性內容」，引導他放進 `manual` 區塊即可（那就不需反吐）。

> 目標：每次手改都回灌 source，使 source 永遠是最新真相，形成可追溯的迭代。

---

## 7. agent-targeting 逃生口（少用）

source 預設是 agent 中立、全部共用。極少數「某段只給特定 agent」的情況，用標記包起來：

```markdown
<!-- agent: claude -->
只有產生 Claude 的設定檔時才納入這段。
<!-- /agent -->

<!-- agent: codex -->
只有產生 Codex 的設定檔時才納入這段。
<!-- /agent -->
```

生成時：只納入「給自己」或「未標記（共用）」的內容，跳過標記給別人的段落。

## 8. 重生 checklist

- [ ] 確認 scope 與輸出路徑正確（§1）
- [ ] 讀完該 scope 所有 source（§2、§3.2）
- [ ] 專案：已讀 `generate.md` 且其覆寫優先於本檔（§3.3）
- [ ] 套用正確 agent profile（§4）
- [ ] 根檔只含 core + 索引，保持精簡（§2、§3.5）
- [ ] 所有子資料夾已正確拆檔到輸出端同名位置，根檔用相對連結指向（§3.6）
- [ ] 既有 `manual` 區塊已原樣保留（§5）
- [ ] 每個生成檔頂部有 banner（§5）
- [ ] 偵測到的擁有區手改已反吐回 source 並記錄理由（§6）
- [ ] 專案輸出沒有重抄全域內容（§1）
- [ ] 已回報產出/保留/反吐摘要（§3.9）
